## 数据格式

通过`Result、PageData、BasePageParam`定义了数据交互的基本格式

::: code-tabs

@tab 基本响应
```js
{
    "code": "00000",
    "message": "成功",
    "data": null
}
```

@tab 分页请求
```js
{
    "pageNum": 1,
    "pageSize": 10
}
```

@tab 分页响应
```js
{
    "code": "00000",
    "message": "成功",
    "data": {
        "total": 20,
        "records": [...]
    },
}
```
:::

## 错误码约定

前后端交互中常用错误码来区分接口是否正常，出现何种错误以便前端区分如何处理。
本项目中的错误码参考《阿里巴巴Java开发手册_嵩山版》中关于错误码的约定，截取部分错误码供参考：

```java
    /**
     * 成功
     */
    SUCCESS("00000", "成功"),

    /**
     * 一级宏观错误码
     */
    USER_ERROR("A0001", "用户端错误"),
    /**
     * 二级宏观错误码
     */
    USER_REGISTRATION_ERROR("A0100", "用户注册错误"),
    /**
     * 二级宏观错误码
     */
    USER_LOGIN_EXCEPTION("A0200", "用户登录异常"),
    USER_ACCOUNT_NOT_EXIST("A0201", "用户账户不存在"),
    USER_ACCOUNT_FROZEN("A0202", "用户账户被冻结"),
    USER_ACCOUNT_DISABLED("A0203", "用户账户已作废"),
    USER_ACCOUNT_EXPIRED("A0204", "用户账户已过期"),
    USER_PASSWORD_ERROR("A0210", "用户名或密码错误"),
    USER_LOGIN_CHECK_ERROR("A0220", "用户身份校验失败"),
    // 一般需要重新登录
    USER_LOGIN_EXPIRED("A0230", "用户登录已过期"),
    USER_LOGIN_REPLACED("A0231", "用户被顶替下线"),
    USER_LOGIN_KICKOUT("A0232", "用户被强制下线"),
```
