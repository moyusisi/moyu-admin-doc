在实际开发中，记录日志是一个常用的功能，本系统中提供了两个与日志记录相关的组件，一个是`打印日志`，另一个是`持久化日志`


## 打印日志

`打印日志`组件提供了`@Log`注解，可以把请求参数等打印到文件或控制台。

### 启用打印日志

在启动类上添加`@EnableCommonLog`注解即可使用。

```java
@EnableCommonLog
@SpringBootApplication
public class SystemApplication {
    public static void main(String[] args) {
        SpringApplication application = new SpringApplication(SystemApplication.class);
        application.run(args);
    }
}
```

### 使用打印日志

在类或方法上添加`@Log`注解即可。若类与方法同时出现，则方法上的注解将覆盖类上的注解

```java
@Log(jsonLog = true)
@RestController
@RequestMapping("/api/sys/org")
public class SysOrgController {

    @Log(jsonLog = true, response = false)
    @PostMapping("/page")
    public Result<PageData<SysOrg>> pageList(@RequestBody SysOrgParam orgParam) {
        ...
    }
}
```

### 注解参数说明

| 参数          | 类型    | 默认值  | 描述                         |
| ------------- | ------- | ------- | ---------------------------- |
| request       | boolean | `true`  | 是否打印请求参数             |
| response      | boolean | `true`  | 是否打印响应结果             |
| runTime       | boolean | `true`  | 是否打印运行时间             |
| jsonLog       | boolean | `false` | 打印日志是否为 json 格式     |
| exceptionOnly | boolean | `false` | 是否只有抛异常时打印请求日志 |

## 持久化日志

`持久化日志`组件提供了`@SysLog`注解，可以把请求参数等日志存储在表`sys_log`中，并且提供了一个日志查询页面（强依赖数据库）。
`持久化日志`组件是以插件的形式提供，如不需要直接在项目的`plugin`中删除即可。

### 使用持久化日志

在方法上添加`@SysLog`注解即可。

```java
    @SysLog(module = "auth", logType = 2, value = "用户登录")
    @PostMapping("/login")
    public Result<String> userLogin(@Validated UserLoginParam loginParam) {
        ...
    }
```

### 注解参数说明

| 参数     | 类型    | 默认值  | 描述                                               |
| -------- | ------- | ------- | -------------------------------------------------- |
| value    | String  | `""`    | 日志标题，对应日志的name                           |
| logType  | int     | `0`     | 日志类型(字典 0默认 1访问日志 2操作日志 3交互日志) |
| module   | String  | `""`    | 系统/模块                                          |
| business | String  | `""`    | 业务, 默认会赋值**短类名**                         |
| operate  | String  | `""`    | 操作, 默认会赋值**方法名**                         |
| content  | String  | `""`    | 内容, 用于自定义记录一些数据                       |
| request  | boolean | `true`  | 是否记录请求参数                                   |
| response | boolean | `false` | 是否记录响应结果                                   |

### 日志查询页面
若项目使用了`持久化日志`插件，则在`系统运维 -> 系统日志`菜单下可以查看日志
