## 数据范围
数据权限控制用户能看到哪些数据，当用户拥有某个菜单权限的前提下，对数据显示的内容进行的限制，与接口的返回数据直接相关，本系统中可以设置每个接口的数据范围，数据范围说明如下：

| 值  | 数据范围       | 说明                                                      |
| --- | -------------- | --------------------------------------------------------- |
| 0   | `不限制`       | 即全部数据，可访问系统所有数据                            |
| 1   | `本人数据`     | 仅可访问本人创建的数据                                    |
| 2   | `本机构数据`   | 仅可以访问本节点 (公司/部门) 的数据，不包括下级节点的数据 |
| 3   | `本机构及以下` | 可以访问本节点 (公司/部门) 及其下级节点的所有数据         |
| 4   | `本公司及以下` | 可访问本节点所属公司节点下的所有数据                      |
| 5   | `自定义`       | 可访问指定节点的数据（需节点列表，查询效率较低）          |

* `本公司及以下`与`本机构及以下`类似，当节点为公司时两者相同；当节点为部门时，会通过组织树向上找到公司节点作为本节点并遍历所有下级节点，常用于许多子公司的集团型公司。

## 同角色不同范围
数据范围是针对每个接口进行控制的，同一个角色中也支持不同接口设置不同的数据范围，系统中通过 `role_has_perm`关系的扩展信息字段来记录，每个权限在与角色关联时，都可单独设置数据范围。


```java title="角色权限关系的数据范围"
// 表sys_relation中的ext_json字段对应结构，表示数据范围信息
public static class ScopeExt {
    /**
     * 数据权限(字典 0无限制 1仅本人数据 2仅本机构 3本机构及以下 4本公司及以下 5自定义)
     */
    private Integer dataScope;
    /**
     * 自定义scope集合
     */
    private List<String> scopeList;
}
```

## 数据范围合并
一个用户同时拥有多个角色时，不同角色针对同一个接口的数据范围设置若有不同，则会产生数据范围冲突。
解决数据范围冲突则需要对不同角色中相同接口的数据范围进行合并，合并时的优先级为： `1仅本人数据 < 2仅本机构 < 3本机构及以下 < 4本公司及以下 < 5自定义 < 0无限制`

具体实现步骤如下：
1. 有无限制则最终为无限制
2. 有自定义则最终为自定义，只是需要两项范围列表合并
3. 其他按照优先级返回大的

代码实现参考项目中的方法：`SysRoleServiceImol#mergeDataScope`

## 核心组件

### 数据范围注解`@DataScope`
需要控制数据权限的接口，必须在控制器层添加`@DataScope`注解，此注解的作用是当用户访问某接口，提取出用户在此接口上的数据权限，并设置到当前用户对象中，以便后续使用。

具体实现在`authZ`模块的`PreDataScopeAspect`中
```
    // @DataScope注解的拦截处理
    public void logHandler(com.moyu.boot.common.core.annotation.DataScope annotation) {
        // 从上下文中获取当前登录用户
        Optional<LoginUser> optUser = LoginUserUtils.getLoginUser();
        if (optUser.isPresent()) {
            LoginUser loginUser = optUser.get();
            // 获取权限标识
            String perm = annotation.value();
            LoginUser.DataScopeInfo dataScopeInfo = loginUser.getPermScopeMap().get(perm);
            if (dataScopeInfo != null) {
                loginUser.setDataScope(dataScopeInfo.getDataScope());
                loginUser.setScopeSet(dataScopeInfo.getScopeSet());
            }
        }
    }
```
### 数据权限插件
获取到当前用户数据范围之后，就可以自行控制数据筛选逻辑了。
数据权限插件由一个注解`@DataPermission`和一个拦截器组成，可直接在`Mapper`层的接口上添加此注解，会自动被拦截器处理。

注解定义如下：
```
/**
 * 数据权限注解(支持指定org或user列名,同时指定则org优先)
 */
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.TYPE, ElementType.METHOD})
public @interface DataPermission {

    /**
     * 用于控制用户过滤的列名，如:create_by
     */
    String userColumn() default "create_by";

    /**
     * 用于控制组织机构过滤的列名，如:org_code
     */
    String orgColumn() default "org_code";
}
```

所有添加`@DataPermission`注解的Mapper接口都会经过拦截器`CustomDataPermissionHandler`对执行的SQL进行处理，其处理逻辑如下表

| 枚举值     | 数据范围       | 处理逻辑                                                                       |
| ---------- | -------------- | ------------------------------------------------------------------------------ |
| ALL        | `不限制`       | 不处理                                                                         |
| SELF       | `本人数据`     | 获取当前用户`username`，在sql后拼接`userColumn = ${username}`                  |
| ORG        | `本机构数据`   | 获取当前用户`orgCode`，在sql后拼接`orgColumn = ${orgCode}`                     |
| ORG_CHILD  | `本机构及以下` | 获取当前用户数据范围列表`scopeSet`，非空时在sql后拼接`orgColumn IN ${orgCode}` |
| COMPANY    | `本公司及以下` | 获取当前用户数据范围列表`scopeSet`，非空时在sql后拼接`orgColumn IN ${orgCode}` |
| ORG_DEFINE | `自定义`       | 获取当前用户数据范围列表`scopeSet`，非空时在sql后拼接`orgColumn IN ${orgCode}` |


## 使用方法
要使用数据权限有两个步骤：
**1. 获取数据范围**
**2. 控制数据筛选**

### 获取数据范围
在控制器层数据范围注解`@DataScope`的作用是**通过权限标识获取访问接口的数据范围**，便于后续流程中可以通过此数据范围控制数据筛选的条件，使用示例如下：

```java title="数据范围"
// 1. Controller层使用数据范围注解@DataScope，通过权限标识sys:user:page获取接口的数据范围
@RestController
@RequestMapping("/api/sys/user")
public class SysUserController {

    /**
     * 分页获取角色列表
     */
    @PreAuthorize("hasAuthority('sys:user:page')")
    @DataScope("sys:user:page")
    @PostMapping("/page")
    public Result<PageData<SysUserVO>> pageList(@RequestBody SysUserParam userParam) {
        PageData<SysUserVO> page = sysUserService.pageList(userParam);
        return Result.success(page);
    }
}
```

### 控制数据筛选
通过获取到的数据范围，控制数据的筛选条件，完成数据的过滤，有两种不同的方式

#### 方式一：注解控制
使用数据权限注解`@DataPermission`控制数据权限的使用示例如下：

```java title="数据筛选"
// 2. Mapper层使用数据权限注解@DataPermission，通过sys_user表中的org_code字段进行组织机构过滤
@Mapper
public interface SysUserMapper extends BaseMapper<SysUser> {

    /**
     * 查询本部门所有的用户
     */
    @DataPermission(orgColumn = "org_code")
    @Select("SELECT * FROM sys_user LIMIT 3")
    List<SysOrg> selectDeptUser(@Param(Constants.WRAPPER) Wrapper<SysOrg> wrapper);
}
```

#### 方式二：编码控制
通过编码控制数据权限的示例如下：

```java
public class SysUserServiceImpl extends ServiceImpl<SysUserMapper, SysUser> implements SysUserService {
    
    public PageData<SysUserVO> pageList(SysUserParam param) {
        // 查询条件
        LambdaQueryWrapper<SysUser> queryWrapper = Wrappers.lambdaQuery(SysUser.class);
        // 指定name查询
        queryWrapper.like(ObjectUtil.isNotEmpty(param.getName()), SysUser::getName, param.getName());
        // 仅查询未删除的
        queryWrapper.eq(SysUser::getDeleted, 0);

        // 获取数据范围，根据不同范围设置不同的筛选条件
        if (!LoginUserUtils.isAdmin()) {
            // 数据范围
            Integer dataScope = LoginUserUtils.getDataScope();
            // 范围列表
            Set<String> scopeSet = LoginUserUtils.getScopes();
            // 控制过滤条件
            if (DataScopeEnum.SELF.getCode().equals(dataScope)) {
                String username = LoginUserUtils.getUsername();
                queryWrapper.eq(SysUser::getCreateBy, username);
            } else if (DataScopeEnum.ORG.getCode().equals(dataScope)) {
                String orgCode = LoginUserUtils.getOrgCode();
                queryWrapper.eq(SysUser::getOrgCode, orgCode);
            } else if (DataScopeEnum.ORG_CHILD.getCode().equals(dataScope)) {
                queryWrapper.in(ObjectUtil.isNotEmpty(scopeSet), SysUser::getOrgCode, scopeSet);
            } else if (DataScopeEnum.COMPANY.getCode().equals(dataScope)) {
                queryWrapper.in(ObjectUtil.isNotEmpty(scopeSet), SysUser::getOrgCode, scopeSet);
            } else if (DataScopeEnum.ORG_DEFINE.getCode().equals(dataScope)) {
                queryWrapper.in(ObjectUtil.isNotEmpty(scopeSet), SysUser::getOrgCode, scopeSet);
            }
            log.debug("数据权限为:{}, 已追加过滤条件", DataScopeEnum.getByCode(dataScope));
        }
        // 分页查询
        Page<SysUser> page = new Page<>(param.getPageNum(), param.getPageSize());
        Page<SysUser> userPage = this.page(page, queryWrapper);
        List<SysUserVO> voList = buildSysUserVOList(userPage.getRecords());
        return new PageData<>(userPage.getTotal(), voList);
    }
}
```
