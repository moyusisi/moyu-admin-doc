## 错误码
前后端交互中常用错误码来区分接口是否正常，出现何种错误以便前端区分如何处理。
通常前后端信息交互有如下场景：
* 场景1. 成功。表示接口正常返回数据。
* 场景2. 出现错误，前端无法处理，需前端进行提示，如前端固定提示"服务器错误，请稍后再试"
* 场景3. 出现错误，前端无法处理，提示信息由后端提供，前端仅进行展示即可。如"用户名已存在"
* 场景4. 出现错误，前端需要区分具体错误种类，然后根据不同错误做不同动作，如登录过期要跳登录页重新登录等。

上述情况中场景4通常需要前后端根据业务实际情况，事先约定好错误码及对应的处理，除此之外的其他情况均可统一处理。

## 错误码定义
根据上述的场景1～3，项目中通过`ExceptionEnum`定义了一些基本的错误码，用最常用、最简单的方式对异常进行区分。

> 0: 成功
> -1: 参数错误。
> 1: 业务异常。用于可预期的但无法继续进行业务操作的异常，常用于
> 2: 系统异常。用于无预期的、未知的异常。
> 3: 自定义异常。用于自定义提示信息的异常，前端可直接展示对应的提示信息。

场景1中，只有错误码0，表示成功。
场景2中，虽然前端处理方式一致，但仍区分了错误码1和2，来区分是否预期内的异常。
场景3中，除了错误码3，单独将参数错误定义为-1表示请求端的错误。

使用时，通过`BaseException`可对错误进行具体描述，描述信息会加工成`参数错误:uid不能为空`
```java
throw new BaseException(ExceptionEnum.INVALID_PARAMETER, "uid不能为空");
```

## 异常定义

基本异常类的定义是`com.moyu.common.exception.BaseException`，它继承了`java.lang.RuntimeException`是一个不受检查异常，并且会减少堆栈信息的打印，只关注于业务信息。
它有三个构造函数：

```java
    // 构造函数一，直接传入错误码和错误信息
    public BaseException(int code, String message) {
        super(message);
        this.code = code;
        this.message = message;
    }
    // 构造函数二，根据枚举创建异常, 以枚举类的描述信息作为异常的message
    public BaseException(ExceptionEnum exceptionEnum) {
        super(exceptionEnum.getMessage());
        this.code = exceptionEnum.getCode();
        this.message = exceptionEnum.getMessage();
    }
    // 构造函数三，补充具体的描述信息, 以枚举+detail作为异常的message
    public BaseException(ExceptionEnum exceptionEnum, String detail) {
        super(exceptionEnum.getMessage());
        this.code = exceptionEnum.getCode();
        String message = exceptionEnum.getMessage();
        if (detail != null && !detail.isEmpty()) {
            message = message + ":" + detail;
        }
        this.message = message;
    }
```
使用时，直接抛出异常即可

```java
    // 要删除的数据 ids
    if (CollectionUtils.isEmpty(ids)) {
        throw new BaseException(ExceptionEnum.INVALID_PARAMETER, "删除失败,未查到指定数据");
    }
```

## 异常处理

在业务代码中抛出的业务异常`BaseException`，通常会使用全局异常处理器`GlobalExceptionHandler`进行处理。当程序抛出异常时，自动捕获这些异常，进行统一处理并返回友好的提示信息，而不是直接崩溃。

```java
@Slf4j
@RestControllerAdvice(annotations = RestController.class)
public class GlobalExceptionHandler {

    /**
     * 异常捕捉处理
     */
    @ExceptionHandler(value = Exception.class)
    public BaseResponse<?> exceptionHandler(Exception e) {
        BaseResponse<?> response = new BaseResponse<>();
        if (e instanceof IllegalArgumentException) {
            // 业务校验中的参数异常
            String message = "参数错误:" + e.getMessage();
            log.error(message);
            response.setCode(ExceptionEnum.INVALID_PARAMETER.getCode());
            response.setMessage(message);
        } else if (e instanceof BaseException) {
            log.error(e.getMessage());
            response.setCode(((BaseException) e).getCode());
            response.setMessage(e.getMessage());
        } else {
            log.error("系统异常", e);
            response.setCode(ExceptionEnum.SYSTEM_ERROR.getCode());
            response.setMessage(ExceptionEnum.SYSTEM_ERROR.getMessage());
        }
        log.info("异常捕捉处理后返回结果为:{}", JSONUtil.toJsonStr(response));
        return response;
    }
}
```
