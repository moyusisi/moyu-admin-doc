::: tip
登录认证就是识别出来"你是谁"，包括：用户认证、颁发令牌、会话管理、用户信息缓存等功能。
:::

## 用户认证
用户认证通过`/auth/login`接口实现，登录接口的流程为: `login接口` -> `用户信息获取` -> `信息检查` -> `生成令牌` -> `缓存信息` -> `返回令牌`

```flow
st=>start: 用户登录
e=>end: 登录成功
op1=>operation: login接口
op2=>operation: 用户信息获取
op3=>operation: 信息检查
op4=>operation: 生成令牌
op5=>operation: 缓存信息
op6=>operation: 返回令牌

op1(right)->op2(right)->op3()->op4(right)->op5(right)->op6(right)
```

## 颁发令牌
颁发令牌需要返回给前端用于后续的权限验证。此令牌不要跟Auth2.0令牌混淆。

通过`TokenService`进行令牌生成，该接口有两种实现，分别是`RedisTokenServiceImpl`和`JwtTokenServiceImpl`，两种令牌的类型可通过配置 `custom.auth.token-type` 进行切换

| 取值    | 说明              |
| ------- | ----------------- |
| `jwt`   | JWT令牌+redis缓存 |
| `redis` | Redis有状态认证   |

两种类型的令牌格式不同，支持的功能并无差异。

## 信息缓存
生成令牌后会在redis中将用户的信息进行缓存，这些信息包括用户基本信息、权限信息、数据范围信息等，可参考`LoginUser`。后续鉴权的过程中通过令牌(token)可以获取这些信息。

## 会话管理
用户登录后，用户信息保存在缓存中，因此可以进行会话管理，如封禁、强制下线等功能
