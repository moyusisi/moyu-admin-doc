## 参数校验
`Bean Validation`是一套基于注解的数据校验规范，如`JSR303、JSR349、JSR380`等，支持`@Null、@NotNull、@Pattern`等，它们位于`javax.validation.constraints`这个包下。

`Hibernate Validator`是对这个规范的实现，并增加了一些其他校验注解，如 `@NotBlank、@NotEmpty、@Length`等，它们位于`org.hibernate.validator.constraints`这个包下。

Spring Boot提供了`spring-boot-starter-validation`模块，已经集成了`Hibernate Validator`，并且Spring验证框架对参数的验证机制提供了`@Validated`注解，对`@Valid`进行了二次封装，支持分组等功能。
更多详情可参考[这里](https://developer.aliyun.com/article/1670620)

## 如何使用
使用时直接在需要校验的参数上添加`@Validated`注解即可，如下所示。

```java  :no-collapsed-lines
    @PostMapping("/add")
    public BaseResponse add(@Validated @RequestBody SysUserParam sysUserParam) {
        ...
    }
    // POJO
    public class SysUserParam {
        /**
         * 姓名
         */
        @NotBlank(message = "用户姓名name不能为空")
        private String name;
        ...
    }
```

## 快速失败
默认情况下，参数校验会把所有规则都检查一遍，若要发现错误立刻失败返回(快速失败模式)，可通过`ValidatorConfig`进行配置：

```java
@Configuration
public class ValidatorConfig {

    @Bean
    public Validator validator() {
        ValidatorFactory validatorFactory = Validation.byProvider(HibernateValidator.class).configure()
                // 快速失败模式
                .failFast(true)
                .buildValidatorFactory();
        return validatorFactory.getValidator();
    }
}
```

## 全局异常处理
项目中应定义一个全局异常处理器`GlobalExceptionHandler`，当程序抛出异常时，自动捕获这些错误，并返回友好的提示信息，而不是直接崩溃。

```java :collapsed-lines=21
@Slf4j
@RestControllerAdvice(annotations = RestController.class)
public class GlobalExceptionHandler {
    /**
     * 异常捕捉处理
     */
    @ExceptionHandler(value = Exception.class)
    public BaseResponse<?> exceptionHandler(Exception e) {
        BaseResponse<?> response = new BaseResponse<>();
        if (e instanceof BindException) {
            // 参数验证机制中，多个字段错误会把所有错误信息拼在一起返回BindException
            BindingResult bindingResult = ((BindException) e).getBindingResult();
            String message = bindingResult.getAllErrors().stream()
                    .map(DefaultMessageSourceResolvable::getDefaultMessage)
                    .collect(Collectors.joining(";"));
            message = "参数错误:" + message;
            log.error(message);
            response.setCode(ExceptionEnum.INVALID_PARAMETER.getCode());
            response.setMessage(message);
        } else if (e instanceof ConstraintViolationException) {
            // 参数验证机制中，违反约束异常
            String message = "参数错误:" + e.getMessage();
            log.error(message);
            response.setCode(ExceptionEnum.INVALID_PARAMETER.getCode());
            response.setMessage(e.getMessage());
        } else if (e instanceof IllegalArgumentException) {
            // 处理 IllegalArgumentException
            String message = "参数错误:" + e.getMessage();
            log.error(message);
            response.setCode(ExceptionEnum.INVALID_PARAMETER.getCode());
            response.setMessage(message);
        } else if (e instanceof ServletRequestBindingException) {
            // ServletRequestBindingException是请求参数绑定到模型属性时出现的异常，如必传参数缺失
            log.error(e.getMessage(), e);
            response.setCode(ExceptionEnum.INVALID_PARAMETER.getCode());
            response.setMessage(ExceptionEnum.INVALID_PARAMETER.getMessage());
        } else if (e instanceof HttpMessageConversionException) {
            // json格式参数进行参数类型转换时，参数转换失败则HttpMessageConversionException
            log.error(e.getMessage(), e);
            response.setCode(ExceptionEnum.INVALID_PARAMETER.getCode());
            response.setMessage(e.getMessage());
        } else if (e instanceof BaseException) {
            log.error(e.getMessage());
            response.setCode(((BaseException) e).getCode());
            response.setMessage(e.getMessage());
        } else {
            log.error("系统异常", e);
            response.setCode(ExceptionEnum.SYSTEM_ERROR.getCode());
            response.setMessage(ExceptionEnum.SYSTEM_ERROR.getMessage());
        }
        log.info("异常捕捉处理后返回结果为:{}", JSONUtil.toJsonStr(response));
        return response;
    }
}
```
