:::

## 参数校验简介
`Bean Validation`是一套基于注解的数据校验规范，如`JSR303、JSR349、JSR380`等，支持`@Null、@NotNull、@Pattern`等，它们位于`javax.validation.constraints`这个包下。

`Hibernate Validator`是对这个规范的实现，并增加了一些其他校验注解，如 `@NotBlank、@NotEmpty、@Length`等，它们位于`org.hibernate.validator.constraints`这个包下。

Spring Boot提供了`spring-boot-starter-validation`模块，已经集成了`Hibernate Validator`，并且Spring验证框架对参数的验证机制提供了`@Validated`注解，对`@Valid`进行了二次封装，支持分组等功能。
更多使用详情可参考[这里](https://developer.aliyun.com/article/1670620)

## 如何使用
使用时直接在需要校验的参数上添加`@Validated`注解即可，如下所示。

```java
    @PostMapping("/add")
    public BaseResponse add(@Validated @RequestBody SysUserParam sysUserParam) {
        ...
    }
    // POJO
    public class SysUserParam {
        /**
         * 姓名
         */
        @NotBlank(message = "用户姓名name不能为空")
        private String name;
        ...
    }
```

## 快速失败
默认情况下，参数校验会把所有规则都检查一遍，若要发现错误立刻失败返回(快速失败模式)，可通过`ValidatorConfig`进行配置：

```java
@Configuration
public class ValidatorConfig {

    @Bean
    public Validator validator() {
        ValidatorFactory validatorFactory = Validation.byProvider(HibernateValidator.class).configure()
                // 快速失败模式
                .failFast(true)
                .buildValidatorFactory();
        return validatorFactory.getValidator();
    }
}
```

## 全局异常处理
项目中应定义一个全局异常处理器`GlobalExceptionHandler`，当程序抛出异常时，自动捕获这些异常，进行统一处理并返回友好的提示信息，而不是直接崩溃。

```java
@Slf4j
@RestControllerAdvice(annotations = RestController.class)
public class GlobalExceptionHandler {
    
    // Spring验证异常(MethodArgumentNotValidException extends BindException)
    @ExceptionHandler(BindException.class)
    public BaseResponse<?> exceptionHandler(BindException e) {
        // 单个字段错误会返回MethodArgumentNotValidException，多个字段错误会把所有错误信息拼在一起返回BindException
        String message = "参数错误:" + e.getBindingResult().getAllErrors().stream()
                .map(DefaultMessageSourceResolvable::getDefaultMessage)
                .collect(Collectors.joining(";"));
        log.error(message);
        return new BaseResponse<>(ExceptionEnum.INVALID_PARAMETER.getCode(), message);
    }

    // Spring校验违反约束异常(ConstraintViolationException extends ValidationException)，如 @Size、@Min、@Max
    @ExceptionHandler(ConstraintViolationException.class)
    public BaseResponse<?> exceptionHandler(ConstraintViolationException e) {
        String message = "参数错误:" + e.getMessage();
        log.error(message);
        return new BaseResponse<>(ExceptionEnum.INVALID_PARAMETER.getCode(), message);
    }
    // ... 其他异常处理
}
```
