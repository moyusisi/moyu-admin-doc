::: tip
本文档以`moyu-admin`为例介绍如何对使用vite构建的vue3项目构建微应用，并介绍微应用的相关配置。
:::

微应用(或子应用)可以是不同的技术栈(Vue、React、Angular)，[qiankun官网](https://qiankun.umijs.org/zh/guide/tutorial#%E5%BE%AE%E5%BA%94%E7%94%A8)有相关的示例，但没有使用vite构建的vue3项目示例，本文档以`moyu-admin`为例介绍如何对使用vite构建的vue3项目构建微应用。

微应用(或子应用)不需要额外安装任何其他依赖即可接入`qiankun`主应用，只需要**暴露相应的生命周期**和**配置打包工具**即可。

## 安装`qiankun`插件
使用vite构建时，为了方便配置，可以安装`vite-plugin-qiankun`插件。

```shell
# 安装 qiankun
npm i qiankun -S
# 安装 qiankun适配vite插件
npm install vite-plugin-qiankun --save-dev
```

## 暴露生命周期
改造子应用的入口文件，暴露 `qiankun` 的生命周期钩子，实现子应用的挂载/卸载逻辑

1. 微应用可以独立运行，因此通过`qiankunWindow.__POWERED_BY_QIANKUN__`判断是否独立运行，独立运行时直接渲染。通过微应用模式运行时，在mount时渲染。
2. 微应用mount时，可以接收主应用传过来的props参数进行一些处理。

```ts title="src/main.ts" 
import { createApp, App as VueApp } from 'vue'
import { renderWithQiankun, qiankunWindow } from 'vite-plugin-qiankun/dist/helper'
...

let vueApp: VueApp | null = null
/**
 * 使用工厂函数创建vue实例，目的是支持微应用模式每次加载子应用得到新实例
 */
const createNewApp = () => {
  const app = createApp(App);
  app.use(createPinia())
  app.use(router)
  ...
  return app
}

// 1. 独立运行时直接渲染
if (!qiankunWindow.__POWERED_BY_QIANKUN__) {
  render()
}

// 封装渲染函数
function render(props: any = {}) {
  const { container } = props
  // 微应用模式：挂载到主应用提供的容器；独立运行：挂载到自身的节点 #app
  const mountNode = container ? container.querySelector('#app') : '#app'
  vueApp = createNewApp()
  vueApp.mount(mountNode!)
}

/********** 微应用模式 **********/

// 2. 微应用模式：导出 qiankun 生命周期
renderWithQiankun({
  // 初始化（只会执行一次）
  bootstrap() {
    console.log('subApp bootstrap...');
  },
  // 挂载（每次切换到微应用时执行）
  mount(props) {
    console.log('subApp mount...', props)
    render(props)
  },
  // 卸载（每次离开微应用时执行）
  unmount() {
    console.log('subApp unmount...');
    if(vueApp) {
      vueApp.unmount()
      vueApp = null
    }
  },
  // 可选，仅使用 loadMicroApp 方式加载微应用时生效
  update(props) {
    console.log('subApp update props...', props);
    return undefined;
  },
})
```

## 配置 Vite 打包工具
1. 注册 qiankun 插件，微应用的名字appName，需与主应用注册的名称一致(必须)
2. 配置跨域，允许qiankun主应用域名跨域访问(推荐)
3. 配置构建选项，输出兼容的资源名称(可选)
4. 定义全局变量，解决qiankun沙箱中window指向问题(可选)


```ts title="vite.config.ts" 
...
import qiankun from 'vite-plugin-qiankun'

// 微应用的唯一标识（需与主应用注册的名称一致）
const appName = 'subApp1'
export default defineConfig(({ mode }): UserConfig => {
  ...
  return {
    ...
    server: {
      ...
      // 2.允许跨域（允许qiankun主应用域名跨域访问）
      cors: {
        origin: [
          'http://localhost:81',   // 主应用开发环境域名
          'https://main-app.com'   // 主应用生产环境域名
        ],
        // 允许携带 cookie（如需）
        credentials: true
      },
      ...
    },
    // 3. 构建配置：输出兼容的资源名称
    build: {
      manifest: true,
      // 生成静态资源的存放路径
      assetsDir: 'assets',
      rollupOptions: {
        output: {
          // 静态资源命名规则
          assetFileNames: 'assets/[name].[hash].[ext]',
          chunkFileNames: 'js/[name].[hash].js',
          entryFileNames: 'js/[name].[hash].js',
          ...
        }
      },
      ...
    },
    build: {
      manifest: true,
      // 生成静态资源的存放路径
      assetsDir: 'assets',
      rollupOptions: {
        output: {
          // 静态资源命名规则
          assetFileNames: 'assets/[name].[hash].[ext]',
          chunkFileNames: 'js/[name].[hash].js',
          entryFileNames: 'js/[name].[hash].js',
          ...
        }
      },
      chunkSizeWarningLimit: 1000
    },
    plugins: [
      vue(),
      // 1. 注册 qiankun 插件
      qiankun(appName, {
        // 开发环境是否开启沙箱（默认 true）
        useDevMode: true
      }),
      ...
    ],

    // 4.定义全局变量：解决 qiankun 沙箱中 window 指向问题
    define: {
      'process.env': process.env,
      '__VUE_OPTIONS_API__': true,
      '__VUE_PROD_DEVTOOLS__': false
    }
  }
})
```

## 路由配置
确保子应用路由在微应用模式和独立运行模式下都能正常工作，避免与主应用路由冲突，微应用模式下运行时需要配置base路径与主应用中的配置对应。

```ts {4-5,8-9} title="src/main.ts" 
import { qiankunWindow } from 'vite-plugin-qiankun/dist/helper'
...

// 微应用模式：base 为微应用路径；独立运行：base 为 '/'
const base = qiankunWindow.__POWERED_BY_QIANKUN__ ? '/subApp1/' : '/'

const router = createRouter({
  // 微应用必须使用 createWebHistory，不能用createWebHashHistory
  history: createWebHistory(base),
  ...
})
```

## 菜单布局配置
为了达到 子应用在**独立运行模式下，能够正常显示侧边栏菜单及头部导航栏**，而在微应用模式下，**被主应用加载时只加载子应用的主内容区**，而不加载子应用的侧边栏菜单及头部导航栏 这样的效果，需要在子应用中根据运行模式配置布局。

```ts {10-15} title="src/store/menu.ts" 
import { qiankunWindow } from 'vite-plugin-qiankun/dist/helper'
...
  // 加载组件
  const loadComponent = (menu) => {
    ...
    if (!component) {
      // 如果没有组件，则将组件设置为 undefined 防止错误加载
      item = undefined;
    } else if (component?.toString() === "Layout") {
      // 微应用模式使用EmptyLayout，独立运行时使用Layout
      if (qiankunWindow.__POWERED_BY_QIANKUN__) {
        item = EmptyLayout
      } else {
        item = Layout
      }
    } else {
      ...
    }
    return item
  }
```
