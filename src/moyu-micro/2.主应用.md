::: tip
本文档以`moyu-admin`为例介绍如何构建主应用，并介绍主应用的相关配置。
:::

主应用不限技术栈，只安装`qiankun`，提供一个挂载微应用的容器，然后注册微应用并`start`即可。

## 安装`qiankun`

```shell
# 安装 qiankun
npm i qiankun -S
```


## 注册微应用
注册微应用(可注册多个)时，需要提供微应用的名称、入口地址、挂载容器和激活规则等信息。
本项目中将注册微应用的代码放在`src/microApp.ts`中，并将启动函数导出，在挂载容器加载完后调用以启动`qiankun`(要防止挂载容器未加载完成就加载子应用)。

若要挂载到`#app`容器中，可将`startQiankun`调用放在`app.mount('#app')`之后即可。


::: code-tabs

@tab 入口文件

```js title="src/main.ts"
import { createApp } from 'vue';
...
import { startQiankun } from './microApp.ts'; // [!code highlight] [!code focus]

const app = createApp(App);
...
// 挂载app
app.mount('#app')

// 启动 qiankun 微前端 // [!code focus]
startQiankun();  // [!code highlight] [!code focus]
```

@tab 微应用注册文件

```ts title="src/microApp.ts"
import { registerMicroApps, start, setDefaultMountApp } from 'qiankun'

// 注册微应用列表
registerMicroApps([
  {
    // 子应用唯一标识：必须与子应用 vite.config.js 中定义的 appName 一致
    name: 'subApp1',
    // 子应用入口地址：
    entry: import.meta.env.MODE === 'dev' ? '//localhost:82/' : '//domain:82/',
    // 主应用中挂载子应用的容器 DOM 节点（需在主应用模板中定义）
    container: '#microApp',
    // 子应用的激活规则：当主应用路由匹配到该路径时，自动挂载子应用
    activeRule: '/subApp1',
    // 可选：传递给子应用的自定义参数（子应用在 mount 生命周期中可接收）
    props: {
      token: localStorage.getItem('TOKEN'),
      userInfo: JSON.parse(<string>localStorage.getItem('USER_INFO')),
    },
  },
]);

// 启动 qiankun 微前端
export const startQiankun = () => {
  start({
    // 可选：开启严格的样式隔离
    sandbox: { experimentalStyleIsolation: true },
    // 可选：禁用预加载（开发环境可关闭，生产环境建议开启）
    prefetch: false,
  });
};
```

:::

## 挂载点
大多数情况，会使用`App.vue`文件中的`#app`作为挂微应用的容器，但这就意味着激活子应用时(`路径为:'/subApp1'`)，主应用的整个页面都将被子应用页面覆盖。在本项目中只希望子应用的内容展现在主应用内容区，因此在布局组件`AppMain`中新增挂载点`#microApp`。
同时子应用挂载时，主应用的router-view将不会渲染，从而实现只展示子应用的内容。

```vue {14-37} title="src/layout/components/AppMain/index.vue" 
<template>
  <div class="admin-ui-main">
    <div id="microApp"></div> <!-- [!code ++] -->
    <router-view v-slot="{ Component, route }" v-if="!microApp"> <!-- [!code highlight] -->
      ...
    </router-view>
  </div>
</template>

<script setup>
import { startQiankun } from "@/microApp.js"; // [!code highlight]
...

// 获取当前路由对象（route 是响应式的，路由变化时会自动更新）
const route = useRoute()
// 是否为子应用
const microApp = ref(false)

// 首次加载会调用onMounted但route不会改变
onMounted(() => {
  console.log('AppMain onMounted...')
  checkMicroApp(route)
  startQiankun()    // [!code ++]
})

// 非首次加载则不再调用onMounted，但route会改变
watch(route, (to) => {
  checkMicroApp(to)
})

// 判断路由是否为子应用路由
const checkMicroApp = (to) => {
  // 定义子应用的路由前缀列表
  const microAppRules = ['/subApp1', '/subApp2']
  // 判断当前路由是否属于子应用
  microApp.value = microAppRules.some(prefix => to.path.startsWith(prefix))
}
...

```

## 注意事项
1. 微应用的真实访问路径就是微应用的入口`entry`，路径最后面的`/`不可省略。
2. 子应用的激活规则`activeRule`不能和主应用的路由冲突。
3. 为了保证样式等静态资源相互隔离，需设置沙箱开启隔离 `sandbox: { experimentalStyleIsolation: true }`
4. 本项目默认`vue-router`的路由模式为`history`模式
5. 因兼容性原因，应用中可能会出现重复路由问题(`qiankun`的bug)。[问题1](https://github.com/umijs/qiankun/issues/1522)、[问题2](https://github.com/umijs/qiankun/issues/1865)
