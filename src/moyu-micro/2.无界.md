::: tip

## 无界微前端
[无界微前端框架](https://wujie-micro.github.io/doc/)利用`iframe`和`webcomponent`来搭建天然的js隔离沙箱和css隔离沙箱，技术栈无关，子应用改造成本极低，在满足跨域条件下子应用甚至可以不改造。

## 主应用
微前端项目由一个主应用和若干个子应用组成，主应用提供一个挂载子应用的容器，将子应用的内容在主应用内渲染。

主应用的改造点有：注册子应用、为子应用提供挂载点、激活子应用(通常是通过路径路由)

### 安装`wujie`

```shell
# vue3 框架
npm i wujie-vue3 -S
```

### 注册子应用
按照[无界](https://wujie-micro.github.io/doc/)官方文档的说明，注册子应用(可注册多个)。
本项目中将子应用注册的代码统一放在文件`mainApp/src/microApp.ts`中，并导出启动函数`startWujie`。
在主应用的入口文件`mainApp/src/main.js`中调用导出的启动函数。


::: code-tabs

@tab 入口文件

```js title="src/main.ts"
import WujieVue from "wujie-vue3"; // [!code focus]
import { startWujie } from './microApp.ts'; // [!code focus]
...

const app = createApp(App);
...
// 挂载app
app.mount('#app')

// 启动 wujie 微前端 // [!code focus]
startWujie();      // [!code highlight] [!code focus]
```

@tab 子应用注册文件

```ts title="src/microApp.ts"
import WujieVue from "wujie-vue3";

const { bus, setupApp, preloadApp, destroyApp } = WujieVue;

// 打印 import.meta.env
console.log('===== 打印 import.meta.env =====', import.meta.env)
console.log('当前环境模式：', import.meta.env.MODE)
console.log('是否为开发环境：', import.meta.env.DEV)

// 子应用的域名(<name,host>)映射
export const microMap: Record<string, string> = {
  "subApp1": import.meta.env.MODE === 'dev' ? '子应用dev环境的地址' : '子应用prod环境的地址',
  "subApp2": import.meta.env.MODE === 'dev' ? '子应用dev环境的地址' : '子应用prod环境的地址',
};

// 默认的子应用周期，可在WujieVue中覆盖
const lifecycles = {
  beforeLoad: (window) => console.log(`${window.__WUJIE.id} beforeLoad 生命周期`),
  // 如果子应用没有做生命周期的改造，那么 beforeMount、afterMount、beforeUnmount、afterUnmount 这四个生命周期将不会调用
  beforeMount: (window) => console.log(`${window.__WUJIE.id} beforeMount 生命周期`),
  afterMount: (window) => console.log(`${window.__WUJIE.id} afterMount 生命周期`),
  beforeUnmount: (window) => console.log(`${window.__WUJIE.id} beforeUnmount 生命周期`),
  afterUnmount: (window) => console.log(`${window.__WUJIE.id} afterUnmount 生命周期`),
  // 子应用保活模式独有
  activated: (window) => console.log(`${window.__WUJIE.id} activated 生命周期`),
  deactivated: (window) => console.log(`${window.__WUJIE.id} deactivated 生命周期`),
  // 子应用加载资源失败后触发
  loadError: (url, e) => console.log(`${url} 加载失败`, e),
};

// 启动 wujie 微前端
export const startWujie = () => {
  const props = {
    "token": localStorage.getItem('TOKEN'),
    "userInfo": JSON.parse(<string>localStorage.getItem('USER_INFO')),
  };
  setupApp({
    // 应用唯一name
    name: "subApp1",
    // 需要渲染的url
    url: microMap["subApp1"],
    // 注入给子应用的属性
    props: props,
    // 预执行
    exec: true,
    ...lifecycles
  });
  setupApp({
    // 应用唯一name
    name: "subApp2",
    // 需要渲染的url
    url: microMap["subApp2"],
    // 注入给子应用的属性
    props: props,
    // 预执行
    exec: true,
    ...lifecycles
  });
};
```

### 挂载子应用
在本项目要达到`上方导航和侧边栏菜单均为主应用的内容，子应用的内容只渲染在内容区`这样的效果，因此把子应用挂载单独放在组件`views/dev/micro/index.vue`中，就像是主应用中的一个普通页面。

```vue title="src/views/dev/micro/index.vue" 
<template>
  <div class="admin-ui-main">
    <!--保活模式，name相同则复用一个子应用实例，改变url无效，必须采用通信的方式告知路由变化 -->
    <WujieVue width="100%" height="100%" :name="subAppName" :url="subAppUrl" :props="props" :sync="false"
              :beforeLoad="beforeLoad" :beforeMount="beforeMount"></WujieVue>
    <a-spin :spinning="subLoading" tip="子应用加载中..." size="large" v-if="subLoading" style="background-color: #f6f8f9" />
  </div>
</template>

<script setup>
import { useRoute } from "vue-router";
import WujieVue from "wujie-vue3";
import { microMap } from "@/microApp.ts";

// 获取当前路由对象（route 是响应式的，路由变化时会自动更新）
const route = useRoute()
// 子应用loading
const subLoading = ref(false);
// 子应用所需参数
const subAppUrl = ref(null);
let subAppName = ref("subApp");
const props = ref({})

// 在mount之前初始化无界所需参数
onBeforeMount(() => {
  console.log('主应用onBeforeMount挂载容器前初始化无界参数')
  initNameUrl()
  initProps()
})

// 初始化wujie子应用参数
function initNameUrl() {
  // 通过 fullPath 自动匹配子应用
  let fullPath = route.fullPath;
  // initName
  for (const appName of Object.keys(microMap)) {
    if (fullPath.startsWith(`/${appName}`)) {
      subAppName.value = appName;
      break;
    }
  }
  // initUrl
  fullPath = fullPath.replace(`/${subAppName.value}`, '');
  let host = microMap[subAppName.value]
  // 兼容host结尾包含'/'于fullPath重复的情况
  if (host.endsWith('/')) {
    host = host.slice(0, -1)
  }
  subAppUrl.value = host + fullPath
}

// 初始化wujie子应用参数
function initProps() {
  props.value = {
    "token": localStorage.getItem('TOKEN'),
    "userInfo": JSON.parse(localStorage.getItem('USER_INFO')),
  }
}

// 子应用生命周期钩子函数
function beforeLoad() {
  console.log(`${subAppName.value} beforeLoad 生命周期`)
  subLoading.value = true;
}
function beforeMount() {
  subLoading.value = false;
  console.log(`${subAppName.value} beforeMount 生命周期`)
}
</script>
```

### 子应用激活入口
在系统的路由中添加路由并使用组件`@/views/dev/micro/index.vue`即可。
当在主应用中访问入口路由时，将会激活子应用，加载子应用对应页面的内容。
如当访问路由`/subApp1/auth/monitor`时，将加载子应用`/auth/monitor`页面的内容，激活入口配置如下（示例为静态路由形式，动态路由也可以）：

```ts {20-25} title="src/router/systemRouter.ts"
import Layout from '@/layout/index.vue'

/**
 * 静态路由
 */
const routes = [
	{
		name: 'layout',
		path: '/',
		component: Layout,
		redirect: '/index',
		meta: { title: '系统菜单', hidden: false, brief: true, icon: 'home-outlined' },
		children: [
			{
				path: '/index',
				name: 'Index',
				component: () => import('@/views/home/index.vue'),
				meta: { title: '首页', icon: 'home-outlined', affix: true }
			},
			{
				path: '/subApp1/auth/monitor',
				name: 'subApp1',
				component: () => import('@/views/dev/micro/index.vue'),
				meta: { title: '子应用', icon: 'home-outlined' }
			}
		]
	},
]

export default routes
```

## 子应用

无界有三种运行模式：单例模式、保活模式、重建模式
其中保活模式、重建模式子应用无需做任何改造工作，单例模式需要做生命周期改造。

本项目为了将子应用页面的局部内容渲染到主应用的内容区，对**动态路由**做了改造，同时也对**生命周期**做了改造

### 跨域改造
子应用的资源和接口的请求都在主应用域名发起，所以会有跨域问题，子应用必须做CORS跨域改造.
在子应用的nginx配置文件中对应的server内添加如下配置即可：

``` sh
    # 允许跨域配置
    # 1. 允许的请求源（*表示允许所有）
    add_header Access-Control-Allow-Origin *;
    # 2. 允许的请求方法
    add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
    # 3. 允许的请求头（*表示所有）
    add_header Access-Control-Allow-Headers '*';
    # 4. 预检请求（OPTIONS）的缓存时间，单位秒，减少OPTIONS请求次数
    add_header Access-Control-Max-Age 3600;
```

### 动态路由改造
普通前端项目`moyu-antdv-admin`中的布局组件`Layout`是包含侧边栏菜单的，为了防止把整个布局都渲染到主应用的内容区，新增一个只包含内容的空布局`EmptyLayout`，对应文件`subApp/src/layout/other/empty.vue`。

```vue title="src/layout/other/empty.vue"
<template>
  <router-view></router-view>
</template>
```

当子应用独立运行时使用`Layout`，当子应用通过`无界`激活被当作子应用运行时使用空布局`EmptyLayout`，因此对动态路由加载文件`subApp/src/store/menu.ts`中加载组件的方法`loadComponent`进行了改造

```ts {13-18} title="src/store/menu.ts"
// 布局组件
import Layout from '@/layout/index.vue'
import EmptyLayout from '@/layout/other/empty.vue'
  ...

  // 加载组件
  const loadComponent = (menu) => {
    let item;
    const component = menu.component
    if (!component) {
      item = undefined;
    } else if (component?.toString() === "Layout") {
      // 微应用模式使用EmptyLayout，独立运行时使用Layout
      if(window.__POWERED_BY_WUJIE__) {
        item = EmptyLayout
      } else {
        item = Layout
      }
    } else {
      item = ...
    }
    return item
  }
```

### 生命周期改造
在子应用的入口文件`subApp/src/main.js`中改造app的生命周，当子应用运行在无界微前端模式下时，单独挂载生命周期


```ts  {38-57} title="subApp/src/main.ts"
import { createApp, App as VueApp } from 'vue'
...

// 无界会在子应用的window对象中注入一些全局变量
declare global {
  interface Window {
    // 是否存在无界
    __POWERED_BY_WUJIE__?: boolean;
    // 原生的window对象
    __WUJIE_RAW_WINDOW__: Window;
    // 子应用沙盒实例
    __WUJIE: { mount: () => void };
    // 子应用mount函数
    __WUJIE_MOUNT: () => void;
    // 子应用unmount函数
    __WUJIE_UNMOUNT: () => void | Promise<void>;
    // 注入对象
    $wujie: {
      bus: any;
      shadowRoot?: ShadowRoot;
      props?: { [key: string]: any };
      location?: Object;
    };
  }
}

let vueApp: VueApp | null = null
/**
 * 使用工厂函数创建vue实例，目的是支持微应用模式每次加载子应用得到新实例
 */
const createNewApp = () => {
  const app = createApp(App);
  app.use(router)
  ...
  return app
}

if (window.__POWERED_BY_WUJIE__) {
  window.__WUJIE_MOUNT = () => {
    vueApp = createNewApp()
    console.log("子应用 mount")
    const props = window.$wujie?.props;
    // 存储token
    if (props?.token) {
      localStorage.setItem('TOKEN', props.token)
    }
    // 存储到userInfo
    if (props?.userInfo) {
      localStorage.setItem('USER_INFO', JSON.stringify(props.userInfo))
    }
    vueApp.mount("#app");
  };
  window.__WUJIE_UNMOUNT = () => {
    console.log("子应用 unmount")
    vueApp?.unmount();
  };
  window.__WUJIE.mount()
} else {
  vueApp = createNewApp()
  console.log("独立运行 mount")
  vueApp.mount("#app");
}
```

## 注意事项
1. 子应用必须做跨域改造
2. 主应用中的激活子应用不同的URl均可以使用同一个组件`micro/index.vue`
