## 环境准备
本项目以`moyu-admin`为基础，所需环境与`moyu-admin`一致，只需要安装`Node`即可。

## 启动项目
改造子应用的入口文件，暴露 `qiankun` 的生命周期钩子，实现子应用的挂载/卸载逻辑

### 拉取代码

```shell
git clone https://github.com/moyusisi/moyu-micro-admin.git
cd moyu-micro-admin
```

### 启动主应用

```shell
# 进入主应用目录
cd mainApp
# 安装依赖
npm install
# 启动项目
npm run dev
```

### 启动子应用

```shell
# 进入子应用目录
cd subApp
# 安装依赖
npm install
# 启动项目
npm run dev
```

### 后端数据
因为接口调用依赖于后端数据，可以使用mock数据进行模拟，或者使用真实数据进行测试。
  
* 若要使用mock数据，需将配置文件中`VITE_API_BASEURL`设置为`""`或`"/"`，不可配置域名，否则会直接请求后端，不经过本地mock服务

若要使用mock数据，可在`/api/sys/userCenter/userMenu`接口返回数据中，"系统运维"目录下增加菜单

```ts title="mock/system/userCenterApi.js"
            // 在系统运维目录下增加子应用菜单
              "children": [
                ...
								{
									"code": "6921c13293123a2d0d7131f0",
									"parentCode": "biz_module",
									"weight": 99,
									"name": "子应用菜单",
									"path": "/subApp1/ops/sys/log",
									"component": "log",
									"meta": {
										"title": "子应用菜单",
										"type": "menu"
									}
								},
              ]
```
